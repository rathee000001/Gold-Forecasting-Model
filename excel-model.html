2. Excel Workbook (Live View)

This is the live workbook view (Raw_Data, Factors_Model, Backtest_Past, Backtest_Future). Scroll inside the embed to view the sheet exactly as in Excel.

2. Sheet: Raw_Data

Raw_Data contains the monthly time-series inputs. This is the single source of truth for the model’s input series before any return calculations or standardization.

What lives here: Gold price (USD) and the factor series used later for return/real-rate/reserve-change construction.

Why it matters: Keeping raw inputs separate prevents accidental mixing of “raw” vs “derived” variables and makes the model fully auditable.

Data flow: Raw_Data feeds Factors_Model (aligned month-by-month).

3. Sheet: Factors_Model

Factors_Model transforms raw series into the variables used for estimation and forecasting. It copies aligned input series from Raw_Data and creates derived monthly variables.

Calculated columns (documented):

  • Gold_Return
  • US_Import_Return
  • USD_Return
  • CPI_Infl_YoY
  • Real_Rate
  • CB_BRICS_Share
  • CB_World_Change
  • CB_BRICS_Change
  • CB_BRICS_Share_Change
  • Optional standardized versions (e.g., Gold_Return_Z)

Data flow: Factors_Model feeds Forecast_Backtest_10Y.

4. Sheet: Forecast_Backtest_10Y

This sheet generates the predicted gold return series using the regression specification and converts predicted returns into a modeled price path for comparison against actual historical prices.

Outputs:

  • Predicted_Return (monthly)
  • Predicted_Price_Path (recursive price series)
  • Actual vs Predicted comparison (for backtesting)

5. Core Equations (as implemented)

The workbook uses a regression framework (with standardized inputs where applicable) to estimate monthly gold returns, then compounds predicted returns into a modeled price path.

Standardization (Z-score): Z(X) = (X − mean(X)) / stdev(X)

Gold_Returnₜ = β₀ + β₁·Real_Rateₜ + β₂·USD_Returnₜ + β₃·US_Import_Returnₜ + β₄·GPR_Indexₜ + β₅·CB_BRICS_Share_Changeₜ + … + εₜ

Predicted monthly return:

Predicted_Returnₜ = β̂₀ + β̂₁·X₁ₜ + … + β̂ₖ·Xₖₜ

Recursive modeled price path:

Predicted_Priceₜ = Predicted_Priceₜ₋₁ × (1 + Predicted_Returnₜ)

Constraints:

  • Linear regression structure
  • Monthly frequency
  • Assumes relationships are sufficiently stable for forecasting/backtesting use
